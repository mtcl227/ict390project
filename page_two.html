Modules Tutorial

So, you've installed Drupal and have heard a lot about these 'modules' that are supposed to make Drupal so dandy. Let's begin by installing a basic module. You can find modules at Drupal's site, or really just by Googling Drupal -module name-. So, search around a bit, read through some basic modules, and grab something you like by right clicking on the tar.gz link for your appropriate Drupal verison (Drupal 7 or Drupal 8, usually) at the end of the module's page and using Copy Link Address. Then, on your Drupal site while logged in as the administrator, navigate to the the Modules tab and click Install new module. Paste the link into the Install from a URL and clicking install. Then follow it up by enabling it from the list, and voila, it's on your site.

That isn't the end though. Next, you need to navigate to the Blocks tab and select a block to place your shiny new module in. For example, if you downloaded a Tetris game module, you might put it in one of the sidebars. Once that's done and you hit save, you'll see the module on your webpage!

Flex Slider

Now let's do something a little more complicated. Search for the Flex Slider module, and install it as described above. However, this time we're not done yet. You'll need the libraries for Flex Slider to work. You'll need to use a file transfer software and find the connection information for your website, including hostname, your admin username and password, and client. Connect via SFTP and you'll see your website has files like Code, Sites, etc.

Now is when you need the libraries. On Flex Slider's webpage, go to the GitHub link and download the .zip file with the libraries in it. Next, you'll need to use your file transfer software to move those libraries into the libraries file of your site. Once that's done, rename the file flexslider.

And then that's it, you're done! You can now use Flex Slider to make carousels of images and other neat things. Some other modules also require this sort of installation, so be sure to keep an eye out for the libraries when trying to get modules to work.
